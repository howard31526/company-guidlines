## Git Commit Message 規範（必遵守）

1. **一律使用英文：繁體中文**撰寫 commit message。
2. 採用以下格式：
   - `英文類別：一句繁體中文話進行描述`
3. 類型僅能使用下列其一（擇一）：
   - `add`、`fix`、`restruct`、`test`、`doc`、
4. 描述需精簡、具體，可讓人看懂「改了什麼」：
   - 建議 10～30 字
   - 不要寫空泛內容（例如：`try`、`change`）


### 範例
- `add：支援 Markdown 條列解析為卡片`
- `fix：分頁超出範圍時回傳空清單並提示`
- `restruct：將分頁邏輯抽成 paginator 模組`
- `test：補齊混中英與空文件案例`
