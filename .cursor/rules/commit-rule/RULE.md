## Git Commit Message 規範（必遵守）

1. **一律使用繁體中文**撰寫 commit message（不得使用英文句子作為主體）。
2. 採用以下格式：
   - `類型：一句話描述`
3. 類型僅能使用下列其一（擇一）：
   - `新增`、`修正`、`重構`、`測試`、`文件`、`維運`、`調整`
4. 描述需精簡、具體，可讓人看懂「改了什麼」：
   - 建議 10～30 字
   - 不要寫空泛內容（例如：`更新`、`修一下`、`改改`）
5. **一個 commit 只做一件事**；若同時包含不同類型的改動，請拆成多個 commit。

### 範例
- `新增：支援 Markdown 條列解析為卡片`
- `修正：分頁超出範圍時回傳空清單並提示`
- `重構：將分頁邏輯抽成 paginator 模組`
- `測試：補齊混中英與空文件案例`
